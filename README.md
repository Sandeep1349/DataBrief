# DataBrief

**AI-powered data analysis platform.** Upload a CSV, Excel, or Parquet file and get an instant dashboard, auto-computed KPIs, interactive charts, an AI chat agent that can query your data in natural language, and a one-click downloadable HTML report — all running locally in Docker.

---

## Features

| | |
|---|---|
| **Upload & Ingest** | Stream-parse CSV, Excel (.xlsx), and Parquet files up to 3 GB into ClickHouse with real-time progress |
| **Auto KPIs** | Total rows, numeric sums/averages, date ranges — computed on ingest and cached |
| **Dashboard** | Time-series chart, category breakdown bar chart, paginated data table |
| **AI Chat** | Three-agent system: Analyst writes and runs ClickHouse SQL, Writer generates narrative, Chat agent handles Q&A with tool use |
| **AI Insights** | One-click executive summary generated by the Writer agent |
| **Report Export** | Download a self-contained HTML report with KPIs, AI narrative, and breakdown table |
| **Persistent Threads** | Chat history stored in ClickHouse; threads survive restarts |
| **Stuck-job Recovery** | Any dataset stuck in `processing` on startup is automatically marked `failed` so you can re-upload |

## Tech Stack

**Backend**
- [FastAPI](https://fastapi.tiangolo.com/) — async Python API with SSE streaming
- [ClickHouse](https://clickhouse.com/) — columnar database for analytics at scale
- [Groq](https://groq.com/) — LLM inference (`meta-llama/llama-4-scout-17b-16e-instruct`)
- Argon2id password hashing, JWT auth
- PyArrow + openpyxl for file parsing

**Frontend**
- [React 18](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
- [Vite](https://vitejs.dev/) dev server
- [Tailwind CSS](https://tailwindcss.com/)
- [Recharts](https://recharts.org/) for interactive charts

**Infrastructure**
- Docker Compose (4 containers: backend, frontend, ClickHouse, CH-UI)
- ClickHouse `ReplacingMergeTree` for mutable metadata (datasets, threads)
- In-process 60-second query cache on the backend

## Architecture

```
┌─────────────────────────────────────────────────────────┐
│  Browser                                                │
│  React + Vite · :5173                                   │
└────────────────────────┬────────────────────────────────┘
                         │ HTTP / SSE
┌────────────────────────▼────────────────────────────────┐
│  FastAPI Backend · :8000                                │
│                                                         │
│  /auth          JWT login                               │
│  /datasets      Upload intake → background pipeline     │
│  /datasets/{id}/analytics   KPIs, charts, report        │
│  /chat          SSE streaming · 3-agent system          │
│                                                         │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐              │
│  │ Analyst  │  │  Writer  │  │  Chat    │              │
│  │ SQL gen  │  │narrative │  │  Q&A     │              │
│  └────┬─────┘  └──────────┘  └──────────┘              │
│       │ Groq API (llama-4-scout)                        │
└───────┼─────────────────────────────────────────────────┘
        │ ClickHouse protocol
┌───────▼─────────────────────────────────────────────────┐
│  ClickHouse · :8123                                     │
│  databrief.datasets  databrief.kpis                     │
│  databrief.chat_threads  databrief.chat_messages        │
│  databrief.<per-dataset-tables>                         │
└─────────────────────────────────────────────────────────┘
```

The background **processing pipeline** (`datasets/processing.py`) streams each uploaded file in chunks, cleans nulls and type-coerces columns, batch-inserts into a per-dataset ClickHouse table, then computes and caches KPIs.

## Quick Start

```bash
# 1. Clone
git clone https://github.com/Sandeep1349/DataBrief.git
cd DataBrief

# 2. Create your .env
cp .env.example .env
```

Open `.env` and fill in:

```env
GROQ_API_KEY=gsk_...          # Required for AI features
CLICKHOUSE_PASSWORD=...        # Any strong password
JWT_SECRET=...                 # Any 32+ character random string
```

Then set your admin password hash:

```bash
# Generates an Argon2id hash and prints it — paste into .env as APP_PASSWORD_HASH=
./databrief.sh hash
```

```bash
# 3. Start everything
./databrief.sh start
```

| Service | URL |
|---------|-----|
| **App** | http://localhost:5173 |
| **API** | http://localhost:8000 |
| **ClickHouse** | http://localhost:8123 |
| **CH-UI** (SQL explorer) | http://localhost:5521 |

Login with `APP_USER` / the password you hashed (default user: `admin`).

## Usage

### Uploading Data

1. Log in at http://localhost:5173
2. Click **Upload Dataset**, pick a CSV / Excel / Parquet file, give it a name
3. A progress bar shows parsing → cleaning → inserting stages
4. Once status is **Ready**, click the dataset to open its dashboard

### Dashboard

- **Overview tab** — KPI cards, time-series chart, category breakdown
- **Data tab** — paginated raw rows with Load More
- **Download Report** — generates a self-contained HTML file with KPIs, AI narrative, and breakdown table

### AI Chat

Click **Chat about this data** from any dashboard, or open the Chat icon in the nav.

- The **Analyst agent** generates and executes ClickHouse SQL to answer data questions
- The **Chat agent** decides when to call the Analyst vs. answer from cached KPIs
- Responses stream token-by-token with "Thinking…" / "Querying…" status events
- Threads are persisted — reload the page and your history is still there

### AI Insights

The **AI Insights** card on the Overview tab auto-loads a 3–5 paragraph executive summary for the dataset using the Writer agent.

The script downloads the Parquet file from the NYC TLC open data endpoint, samples 20k rows, and uploads them via the DataBrief API. After it completes, the dataset appears in the dashboard with 19 columns including fare amounts, trip distances, pickup/dropoff timestamps, and location IDs.

## ClickHouse Schema

Every uploaded dataset gets its own table (`databrief.<uuid>`) with columns inferred from the file schema. The core metadata tables are:

| Table | Engine | Purpose |
|-------|--------|---------|
| `datasets` | `ReplacingMergeTree(updated_at)` | Dataset metadata and status |
| `dataset_progress` | `MergeTree` | Processing progress events |
| `kpis` | `ReplacingMergeTree` | Cached KPIs per dataset |
| `chat_threads` | `ReplacingMergeTree(updated_at)` | Conversation threads |
| `chat_messages` | `MergeTree` | Immutable message history |

`ReplacingMergeTree` tables are read with `FINAL` to get the latest version of each row without background merges racing ahead.

## AI Agent System

The three-agent system uses the Groq API (`llama-4-scout-17b-16e-instruct`):

```
User question
      │
      ▼
 Chat Agent  ──── KPIs in context? ──► answer directly
      │
      │ no — needs live data
      ▼
 Analyst Agent
  • generates a ClickHouse SELECT query
  • enforces SELECT-only, LIMIT ≤ 500, no CTEs/window functions
  • executes query, returns rows
      │
      ▼
 Chat Agent  ──► synthesizes final answer with query results
      │
      ▼
 SSE stream to browser (thinking events + text chunks)
```

The **Writer Agent** is invoked separately for the Insights card and the HTML report to generate a 3–5 paragraph executive summary from KPIs.

SQL safety rules enforced server-side: must start with `SELECT`, no statement separators (`;`), `LIMIT` injected/capped at 500 rows, no table mutation keywords.

## Development

To rebuild a single service after a code change:

```bash
docker compose restart backend   # Python changes hot-reload via uvicorn --reload
docker compose restart frontend  # Vite HMR handles most frontend changes automatically
```

To wipe all data and start fresh:

```bash
./databrief.sh reset
./databrief.sh start
```

```bash
./databrief.sh logs backend
```
